## 2. Quantization Techniques

### GGUF Quantization (llama.cpp)

The standard for local deployment. Recommended levels for 2026:

| Level | Quality Loss | Size Reduction | Use Case |
|-------|-------------|----------------|----------|
| Q8_0 | Minimal | 2x | Quality-critical applications |
| **Q5_K_M** | Low | 3x | **Best balance (recommended)** |
| **Q4_K_M** | Low-moderate | 4x | Limited VRAM / CPU-only |
| Q3_K_M | Moderate | 5x | Memory-critical scenarios |
| Q2_K | Noticeable | 8x | Last resort |

**Pro tip:** Use importance matrix quantization for better quality:
```bash
./llama-imatrix -m model.gguf -f calibration_data.txt -o imatrix.dat
./llama-quantize --imatrix imatrix.dat model.gguf model-q4.gguf Q4_K_M
```

### BitNet: 1.58-bit Ternary Quantization

Microsoft's BitNet is the most aggressive quantization approach, using weights of only {-1, 0, +1}:

- **Memory reduction:** 16–32x vs full-precision
- **Energy on x86:** 82% reduction
- **Speed on ARM:** 1.37x–5.07x faster than comparable approaches
- **Speed on x86 vs llama.cpp:** 2.37x–6.17x faster
- **Accuracy:** BitNet b1.58 2B4T (trained on 4T tokens) matches full-precision models at same parameter count

**Critical distinction:** BitNet trains natively with ternary weights, rather than quantizing post-training. This lets the network adapt its representations to the constraint.

**Current working model:** BitNet b1.58 2B4T (1.3GB download, 1.1GB RAM at runtime, ~27 tok/s on 8-thread CPU)

**Sparse-BitNet (March 2026):** Microsoft Research published work showing 1.58-bit LLMs are naturally friendly to semi-structured sparsity, enabling further efficiency gains.

## 3. llama.cpp Developments (2026)

llama.cpp remains the default runtime for local inference, with 105K+ GitHub stars. Key 2026 developments:

### Recent Releases
- **b8858 (April 20, 2026):** Optimized x86 and generic CPU q1_0 dot product operations — hand-tuned assembly for AVX2/AVX-512
- **b8814 (April 16, 2026):** Added 128-bit RVV (RISC-V Vector) implementation for quantization vector dot products
- **b8608 (April 1, 2026):** Refactored llama_model internals
- **b8145 (Feb 24, 2026):** LFM2-24B-A2B model support updates
- **b8099 (Feb 19, 2026):** PowerPC FP16 MMA path for quantization

### CPU Optimization Trends
- **Hand-optimized kernels:** PR #21636 (pl752) — manual assembly for q1_0 dot products on x86, closing the gap between local and cloud GPU performance
- **ARM NEON:** Optimized NVFP4 dot products with LUT + vpaddq + vfmaq (Feb 2026)
- **SVE (Scalable Vector Extensions):** Vector-length agnostic implementations for 512-bit, 256-bit, and 128-bit vector lengths
- **AVX512-FP16:** Native support for F16 operations merged (PR #20529)

### Key Insight
Memory bandwidth, not raw compute, is the constraint for large models. Quantization shrinks data movement; optimized kernels ensure arithmetic completes without wasted cycles. Together, these software improvements frequently outpace hardware upgrades on a cost-per-performance basis.

## 5. Deployment Options

### Ollama (Easiest)
```bash
ollama pull qwen3.5:4b
ollama pull phi:3.8b
ollama pull gemma3:4b
ollama pull llama3.2:3b
```

### llama.cpp (Most Optimized)
```bash
git clone https://github.com/ggml-org/llama.cpp
cd llama.cpp && make -j
./llama-quantize model.gguf model-q5.gguf Q5_K_M
./llama-cli -m model-q5.gguf -p "Hello"
```

### MLX (Apple Silicon)
20–50% faster than llama.cpp on Apple Silicon. Best for M-series Macs.

### WebLLM (Browser)
Runs entirely in browser with WebGPU. 80% of native performance, no server required.

## 6. Key Takeaways for Local Deployment

1. **Small models are now genuinely capable** — Qwen 3.5-4B and Phi-4-mini outperform last-generation models 5–18x their size
2. **Q5_K_M is the sweet spot** for quantization — best balance of quality and size
3. **BitNet enables 100B parameter inference on a single CPU** — ternary quantization is not a gimmick, it's a paradigm shift
4. **llama.cpp software improvements outpace hardware** — hand-optimized kernels deliver gains no CPU upgrade can match
5. **3–4B models fit on any 8GB GPU** with Q4 quantization, making local inference accessible to almost everyone
6. **Hybrid architectures are best practice** — route 95% of queries to local SLMs, 5% to cloud LLMs for complex tasks
