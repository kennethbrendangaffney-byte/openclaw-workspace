## 🛡️ Linux Hardening Checklist (2026)

### 1. System Updates & Patching
- [ ] Enable **unattended-upgrades** for security updates only
- [ ] Subscribe to distribution security mailing list (ubuntu-security-announce)
- [ ] Run `apt list --upgradable` weekly
- [ ] Reboot after kernel updates (check `needs-restart` or `/var/run/reboot-required`)

### 2. SSH Hardening
- [ ] Disable root login: `PermitRootLogin no`
- [ ] Disable password auth: `PasswordAuthentication no` (use keys only)
- [ ] Change default port (security through obscurity + log noise reduction)
- [ ] Use `AllowUsers` or `AllowGroups` to restrict access
- [ ] Install `fail2ban` or `sshguard` for brute-force protection
- [ ] Consider `AllowUsers karen` only

### 3. Firewall (UFW / iptables / nftables)
- [ ] Default deny incoming: `ufw default deny incoming`
- [ ] Allow only necessary ports (SSH, maybe HTTP/HTTPS if serving)
- [ ] Enable logging: `ufw logging on`
- [ ] Rate-limit SSH: `ufw limit 22/tcp`

### 4. User & Permission Hardening
- [ ] No unnecessary sudo access — use `sudo` group sparingly
- [ ] Set strong password policy (even if using keys): `pam_pwquality`
- [ ] Review `~/.ssh/authorized_keys` regularly
- [ ] Disable unused accounts: `sudo passwd -l username`

### 5. Filesystem & Encryption
- [ ] Full disk encryption (LUKS) — already standard on Ubuntu install
- [ ] Encrypt sensitive data at rest with `gocryptfs` or `encfs`
- [ ] Separate `/home` partition for easier reinstallation
- [ ] Enable `noexec`, `nodev`, `nosuid` on `/tmp` and `/var/tmp` if separate mounts

### 6. Application Sandboxing
- [ ] Use **Flatpak** or **Snap** for untrusted applications (sandboxed by default)
- [ ] Consider **Firejail** for browser and other internet-facing apps
- [ ] Run services as unprivileged users (never root)
- [ ] Use **systemd** hardening options: `PrivateTmp=yes`, `NoNewPrivileges=yes`, `ProtectSystem=strict`

### 7. Audit & Monitoring
- [ ] Install `auditd` for system call auditing
- [ ] Enable `sysstat` / `sar` for performance baseline
- [ ] Review logs regularly: `/var/log/auth.log`, `/var/log/syslog`
- [ ] Set up `logwatch` or `aide` for intrusion detection
- [ ] Consider **Wazuh** or **OSSEC** for host-based IDS (self-hosted)

### 8. Network Security
- [ ] Disable IPv6 if not used (reduces attack surface)
- [ ] Use **DNS-over-HTTPS (DoH)** or **DNS-over-TLS (DoT)** to prevent ISP snooping
- [ ] Install **Pi-hole** or **AdGuard Home** for network-level ad/tracker blocking
- [ ] Use **Tailscale** or **WireGuard** for secure remote access (avoid port-forwarding)

## 👤 Personal OPSEC Guide

### Identity & Accounts
1. **Use unique emails per service** — SimpleLogin or AnonAddy for aliases
2. **Hardware 2FA** — YubiKey or Nitrokey for critical accounts (Google, GitHub, banking)
3. **Separate "personas"** — Work email, personal email, shopping email, social email
4. **Freeze credit reports** — Prevents identity theft-based financial fraud
5. **Use fake answers for security questions** — Treat them like passwords (store in vault)

### Device Hygiene
1. **Full disk encryption** — LUKS on Linux, FileVault on Mac, BitLocker on Windows
2. **Automatic screen lock** — 5 minutes max, require password on wake
3. **Disable Bluetooth when not needed** — BlueBorne-style attacks still relevant
4. **Cover webcam** when not in use — Low-tech but effective
5. **Use a privacy screen** in public spaces

### Browser & Online
1. **Firefox + uBlock Origin + Privacy Badger** — Baseline
2. **Multi-account containers** — Isolate Facebook, Google, shopping from main browsing
3. **Tor Browser** for sensitive research — Not for daily use (fingerprinting)
4. **Disable WebRTC** — Prevents IP leaks behind VPN
5. **Use `about:config` hardening** or **Arkenfox user.js**

### Communication OPSEC
1. **Assume all SMS is compromised** — Use Signal for anything sensitive
2. **Verify Signal safety numbers** — After initial contact or key change
3. **Disappearing messages** — Enable for sensitive conversations
4. **No sensitive talk on employer devices** — They own that data
5. **Be paranoid about "urgent" requests** — Social engineering is the #1 attack vector

### Physical Security
1. **Don't leave devices unattended** in public — Even "just for a minute"
2. **Use a Kensington lock** at co-working spaces / libraries
3. **Shred sensitive documents** — Cross-cut shredder
4. **Be aware of shoulder surfing** — Especially on trains, cafes
5. **Don't discuss sensitive topics in public** — Assume someone is listening
