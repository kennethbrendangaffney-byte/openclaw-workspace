## 1. Ollama (v0.21.0 — April 16, 2026)

Ollama remains the dominant local LLM orchestration platform. The latest release (v0.21.0) landed just days ago with significant additions:

- **Gemma 4 on MLX:** Full support for Google's Gemma 4 via MLX on Apple Silicon, including a text-only MLX runtime. Added mixed-precision quantization, better capability detection, and new op wrappers (Conv2d, Pad, activations, trig, masked SDPA, RoPE-with-freqs).
- **GitHub Copilot CLI integration** in `ollama launch` alongside other coding agents.
- **OpenCode inline config** — matches how other integrations are handled.
- **ROCm 7.2.1** support update.
- **macOS cross-compile fixes** and quieter cgo builds.

### Release Cadence
Very active: v0.20.5 (Apr 9) → v0.20.6 (Apr 12) → v0.20.7 (Apr 13) → v0.21.0 (Apr 16). Rapid iteration on Gemma 4 support and MLX backend.

### Notable Ollama-Adjacent Projects

- **Lemonade:** Community-built local AI server with NPU/GPU optimizations (AMD Ryzen AI, Radeon, Strix Halo). Supports GGUF, FLM, ONNX models. Has mobile apps (iOS/Android) and marketplace integrations.

## 3. BitNet / 1.58-bit Quantization

Microsoft's BitNet remains the most radical quantization approach — ternary weights (-1, 0, +1) at 1.58 bits.

### Current State
- **Primary model:** `microsoft/bitnet-b1.58-2B-4T` — 2B parameters, ~1.3GB download, 1.1GB RAM at runtime.
- **HuggingFace downloads:** 69,416 (GGUF version), 15,718 (original), 6,101 (BF16).
- **Community models:** 1bitLLM has larger variants (3B, large, xl), plus OLMo-Bitnet-1B from NousResearch.

### Recent Optimizations (Microsoft BitNet repo)
- **Parallel kernel implementations:** Weight & activation parallel processing in W2A8 vet_dot kernel.
- **Native I2_S GEMM/GEMV:** Full integration into ggml/llama.cpp architecture.
- **Configurable tiling:** Block sizes and parallelism levels tunable per CPU architecture.
- **Embedding quantization:** Q6_K format recommended — best balance of memory, speed, and perplexity.
- **Performance:** Up to ~2x speedup over original implementation on AMD EPYC with activation parallel strategy.

### Community Innovation
- **R3-Engine:** Zero-Copy 1.58-bit engine in 100% Safe Rust. Claims 80-117 tok/s single-threaded on Ryzen 9950X3D via AVX-512. Streams weights from SSD via mmap. Also compiles to WebAssembly for in-browser inference. **Status:** Architecture complete, but currently outputs `<unk>` — activation/logit sampling needs debugging.
- **dlgo:** Pure Go inference engine with Vulkan GPU backend. Claims to beat Ollama's CUDA performance on some workloads. Zero dependencies — no Python, no CUDA.
- **26x speedup PR:** Community contribution using AVX-512 and 2-bit encoding for BitNet sparse ops.

### Our Setup
We have BitNet 2B working locally at ~27 t/s on CPU (8 threads). 8B models remain problematic — TL1 format needs kernel rebuild, I2_S conversions have issues. The 2B model is the largest reliable BitNet option currently.

## 4. New Inference Engines & Tools

## 6. Quantization Techniques

### GGUF Ecosystem (llama.cpp / Ollama)
- **Q4_K_M / Q5_K_M:** Still the sweet spot for quality vs size
- **Q6_K:** Best for embedding quantization (BitNet recommendation)
- **Q8_0:** Near-lossless for critical applications
- **Mixed-precision:** Ollama's MLX backend now supports mixed-precision quantization

### BitNet / 1.58-bit
- **I2_S format:** Native ternary (-1, 0, +1) — 1.58 bits per weight
- **Q6_K embeddings:** Recommended for BitNet models to balance speed/quality
- **Limitation:** Only 2B models work reliably; larger sizes have format compatibility issues

### Other Approaches
- **ONNX:** Supported by Lemonade for cross-platform deployment
- **FLM:** Lemonade's native format

## 8. Key Takeaways for Our Setup

1. **BitNet 2B is still our best local option** for raw efficiency, but the ecosystem hasn't cracked larger BitNet models yet.
2. **Ollama v0.21.0** is worth keeping an eye on — Hermes agent and Gemma 4 support are interesting, but we don't run Apple Silicon.
3. **llama.cpp** remains the gold standard engine — daily builds, widest hardware support.
4. **dlgo** is intriguing as a zero-dependency alternative — pure Go, Vulkan GPU, no Python stack.
5. **R3-Engine** is ambitious (Rust, zero-copy, Wasm) but not yet generating coherent output.
6. **Small models (2B-4B)** are getting surprisingly capable — Gemma 4 E2B, Qwen3.5-0.8B, etc.
7. **Quantization sweet spot:** Q4_K_M for general use, Q6_K for embeddings, I2_S for BitNet.
