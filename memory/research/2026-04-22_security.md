**Research focus:** Latest cybersecurity threats, Linux hardening techniques, privacy tools, and personal OPSEC for practical self-hosted security.

## 1. Current Threat Landscape (2026)

### Key Statistics
- **79%** of Linux attacks use **no malware** — attackers "live off the land" using built-in tools (bash, cron, curl)
- **5,530** kernel CVEs filed in 2025 — highest annual count on record
- **89%** of Linux endpoint attacks involve SSH credential stuffing or brute force
- **96%** of top 1M web servers run Linux — making it a prime target

### Top 5 Linux Threats in 2026

1. **SSH Brute Force & Credential Stuffing**
   - Fresh VPS receives first probing attempt within **90 seconds** of going live
   - CVE-2025-26465: OpenSSH MITM vulnerability during client auth
   - **Mitigation:** Disable password auth, use Ed25519 keys, change default port, deploy Fail2Ban/CrowdSec

2. **Privilege Escalation CVEs**
   - CVE-2025-6018/6019: udisks daemon chain → reliable local root escalation
   - eBPF subsystem increasingly targeted (enabled by default, accessible to unprivileged users)
   **Mitigation:** Patch aggressively, audit sudoers, enable SELinux/AppArmor

3. **Cryptojacking**
   - Silent XMRig miners deployed via cron, CPU throttled to 70-80% to avoid detection
   - IoT devices (routers, NAS, cameras) recruited into botnets at scale
   **Mitigation:** Monitor CPU/network, audit crontabs, check for deleted executables in /proc

4. **Supply Chain Attacks**
   - Post-XZ Utils backdoor era: compromised container images, typosquatted PyPI/npm packages
   - **New in 2025:** AI-generated package injection — attackers exploit LLM hallucinations, register non-existent package names with malicious payloads
   **Mitigation:** Verify GPG signatures, pin container digests (never use :latest), use Falco/Trivy/Cosign

5. **Kernel Rootkits**
   - CVE-2024-1086 ("Flipping Pages") actively weaponized by RansomHub, Akira
   - Native Linux ransomware variants: LockBit 5.0, Qilin — targeting VMware ESXi
   **Mitigation:** AIDE integrity monitoring, rkhunter/chkrootkit, run from trusted baseline

## 2. Linux Hardening Best Practices (2026)

### Critical First Steps
| Priority | Action | Command/Config |
|----------|--------|----------------|
| 1 | Disable SSH password auth | `PasswordAuthentication no`, `PermitRootLogin no` |
| 2 | Use Ed25519 key pairs | `ssh-keygen -t ed25519` |
| 3 | Enable firewall (default deny) | `ufw default deny incoming` |
| 4 | Automate security updates | `unattended-upgrades` |
| 5 | Enable AppArmor/SELinux | `aa-enforce` profiles |

### SSH Hardening
```bash
# /etc/ssh/sshd_config
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
Port 2222              # non-default
MaxAuthTries 3
LoginGraceTime 20
```

### Kernel Hardening (sysctl)
```bash
# IP spoofing protection
net.ipv4.conf.all.rp_filter = 1

# Ignore ICMP redirects
net.ipv4.conf.all.accept_redirects = 0

# Disable source routing
net.ipv4.conf.all.accept_source_route = 0

# Enable SYN cookies
net.ipv4.tcp_syncookies = 1

# Ignore ping requests (optional)
net.ipv4.icmp_echo_ignore_all = 1
```

### File System Security
```bash
# Secure /tmp mounts
/tmp     tmpfs   defaults,noexec,nosuid,nodev   0 0

# Find world-writable root files
find / -xdev -type f -perm -0002 -user root 2>/dev/null

# Find setuid/setgid binaries
find / -xdev \( -perm -4000 -o -perm -2000 \) -type f 2>/dev/null
```

### Essential Monitoring Commands
```bash
# Check for brute force attempts
grep "Failed password" /var/log/auth.log | wc -l

# Spot high-CPU processes
ps aux --sort=-%cpu | head -15

# Unexpected network connections
ss -tunap | grep ESTABLISHED

# Audit all crontabs
for user in $(cut -f1 -d: /etc/passwd); do crontab -u $user -l 2>/dev/null; done

# Check for deleted executables (memory-resident malware)
ls -la /proc/*/exe 2>/dev/null | grep deleted

# Recent system file modifications
find /etc /usr /bin /sbin -mtime -1 -type f 2>/dev/null
```

## 3. Recommended Self-Hosted Security Stack

### Intrusion Prevention
| Tool | Role | Notes |
|------|------|-------|
| **CrowdSec** | Collaborative IPS | 60x faster than Fail2Ban, Go-based, community threat intel |
| **Fail2Ban** | Log-based banning | Classic, simple, perfect complement to CrowdSec |

### Network Security
| Tool | Role | Notes |
|------|------|-------|
| **WG-Easy** | WireGuard VPN UI | Web UI for WireGuard, QR code client creation |
| **UFW/Firewalld** | Firewall | Default-deny inbound, rate-limit SSH |

### Secrets & Access
| Tool | Role | Notes |
|------|------|-------|
| **Vaultwarden** | Password manager | Self-hosted Bitwarden, ~50MB RAM, AES-256 + Argon2 |
| **HashiCorp Vault** / **OpenBao** | Secrets management | Dynamic credentials, audit logging, encryption-as-a-service |

### Monitoring & Integrity
| Tool | Role | Notes |
|------|------|-------|
| **AIDE** | File integrity monitoring | Baseline from clean install, detect drift |
| **Lynis** | Security auditing | CIS benchmark alignment, actionable reports |
| **Wazuh** | SIEM/XDR | Centralized logging, anomaly detection |
| **Falco** | Runtime security | Container/Kernel threat detection |

### Container Security
| Tool | Role | Notes |
|------|------|-------|
| **Trivy** | Vulnerability scanner | Scan images, filesystems, repos |
| **Cosign** | Signature verification | Sign and verify container images |

## 4. Privacy Tools & Personal OPSEC

### Communication
- **Signal** — E2EE messaging (gold standard)
- **SimpleX Chat** — No user IDs, no contact discovery server
- **Session** — Decentralized, no phone number required

### Browsing
- **Firefox** + uBlock Origin + Multi-Account Containers
- **Tor Browser** — For anonymity-critical activities
- **Brave** — Chromium-based with built-in blocking (trade-off: crypto baggage)

### Search
- **DuckDuckGo** / **SearXNG** (self-hosted meta-search)
- **Startpage** — Google results without tracking

### DNS
- **Pi-hole** / **AdGuard Home** — Network-level blocking
- **Unbound** + **DNS-over-TLS** — Recursive resolver, no upstream logging

### File Storage
- **Nextcloud** (self-hosted) — E2EE optional, full control
- **Cryptomator** — Client-side encryption for any cloud
- **Syncthing** — P2P sync, no central server

### Email
- **Proton Mail** — E2EE, Swiss jurisdiction
- **Tutanota** — German, open source clients
- **SimpleLogin** / **AnonAddy** — Email aliasing for OPSEC

### Password Hygiene
- **Vaultwarden** (self-hosted) — Never reuse passwords
- **2FA everywhere** — Hardware keys (YubiKey) > TOTP > SMS
