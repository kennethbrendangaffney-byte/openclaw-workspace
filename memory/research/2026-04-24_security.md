## 🛡️ Linux Hardening — Essential Checklist

### 1. SSH Lockdown (Critical)
- **Disable root login:** `PermitRootLogin no`
- **Key-only auth:** `PasswordAuthentication no`, use ED25519 keys
- **Restrict users:** `AllowUsers deployer`
- **Strong algorithms only:**
  - Kex: `curve25519-sha256`, `sntrup761x25519-sha512`
  - Ciphers: `chacha20-poly1305`, `aes256-gcm`
  - MACs: `hmac-sha2-512-etm`, `umac-128-etm`
- **Idle timeout:** `ClientAliveInterval 300`, `ClientAliveCountMax 2`
- **Optional:** Change default port, disable X11/TCP/agent forwarding

### 2. Firewall — Default Deny
- **UFW (Ubuntu):**
  ```bash
  sudo ufw default deny incoming
  sudo ufw default allow outgoing
  sudo ufw allow ssh
  sudo ufw allow 80/tcp
  sudo ufw allow 443/tcp
  sudo ufw limit ssh  # rate limit
  sudo ufw enable
  ```
- **nftables** for advanced setups: rate limiting per IP, geo-blocking, port knocking

### 3. Automated Updates
- **Ubuntu/Debian:** `unattended-upgrades` with security-only origins
- **Enable auto-reboot** at maintenance window (e.g., 3 AM)
- **Monitor CVE feeds:** Ubuntu Security Notices, Red Hat Advisories

### 4. Brute Force Protection
- **Fail2Ban:** Ban IPs after 3 failed attempts, progressive bans for repeat offenders
- **UFW rate limiting:** `ufw limit ssh` (max 6 connections per 30s)
- **nftables dynamic sets:** Per-IP connection rate limiting

### 5. Mandatory Access Control (MAC)
- **AppArmor (Ubuntu):** `aa-status`, `aa-enforce` for nginx, services
- **SELinux (RHEL):** Enforcing mode, use `audit2allow` for custom policies
- **Never disable MAC** — learn to work with it

### 6. Kernel Hardening (`sysctl`)
```bash
# /etc/sysctl.d/99-security.conf
net.ipv4.conf.all.rp_filter = 1          # IP spoofing protection
net.ipv4.conf.all.accept_redirects = 0   # Ignore ICMP redirects
net.ipv4.conf.all.send_redirects = 0
net.ipv4.conf.all.accept_source_route = 0
net.ipv4.tcp_syncookies = 1              # SYN flood protection
net.ipv4.conf.all.log_martians = 1       # Log suspicious packets
net.ipv4.icmp_echo_ignore_all = 1        # Ignore ping (optional)
```

### 7. File Integrity Monitoring
- **AIDE:** Initialize database, run daily checks via cron, email on changes
- Detects: unauthorized config changes, new binaries, backdoors

### 8. Service Hardening
- **Audit running services:** `systemctl list-units --type=service --state=running`
- **Disable unnecessary:** cups, avahi-daemon, bluetooth, rpcbind (if unused)
- **Service accounts:** Run apps under dedicated users with `NoNewPrivileges`, `ProtectSystem=strict`, `PrivateTmp`
- **File permissions:** `chmod 600` for sshd_config, shadow; audit SUID/SGID binaries

### 9. Disk & Boot Security
- **LUKS encryption** for data at rest (enable during install)
- **GRUB password** to prevent single-user mode access
- **Secure `/tmp`, `/var/tmp`, `/dev/shm`:** `noexec,nosuid,nodev`

### 10. Logging & Monitoring
- **Centralized logging:** Ship to remote syslog/SIEM
- **Key logs:** `/var/log/auth.log`, `/var/log/syslog`, `/var/log/kern.log`
- **Prometheus + Grafana** for metrics
- **Lynis** for security auditing and scoring
