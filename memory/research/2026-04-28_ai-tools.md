## 🏠 Local / Self-Hosted AI

| Tool | What It Does | Best For |
|------|-------------|----------|
| **Ollama** | Run LLMs locally with `ollama run modelname`. Handles download, quantization, GPU/CPU allocation. OpenAI-compatible API at localhost:11434. | Quick local setup, experimentation, personal use. Best entry point. |
| **vLLM** | Production-grade LLM serving. PagedAttention for efficient memory use. High throughput. | Production deployments, high-traffic APIs. |
| **LM Studio** | GUI for running local LLMs. Model management, chat interface, API server. | Users who want a graphical interface rather than CLI. |
| **Open WebUI** | Self-hosted web interface for Ollama/LLMs. RAG support, code completion, multi-user. | Team/shared local AI with web frontend. |
| **SGLang** | Efficient LLM serving with structured generation support. | Structured output needs (JSON, schemas). |
| **LocalAI** | Drop-in OpenAI API replacement running locally. | Migrating from OpenAI API to local without code changes. |
| **Llamafile** | Single-file executable LLMs. No dependencies. | Portable, offline AI — run from a USB stick. |
| **TGI (Text Generation Inference)** | HuggingFace's production inference server. | Enterprise deployments with HuggingFace ecosystem. |

**Recommended local stack for 2026:**
1. **Ollama** — for running models (Llama 4, Mistral, Gemma, DeepSeek, Qwen 2.5/3.5)
2. **Open WebUI** — for web interface, RAG, and team access
3. **Docker** — for containerized deployment
4. **Optional: vLLM** — if you need production-grade throughput

**Key 2026 developments:**
- **Llama 4** now available locally via Ollama
- **DeepSeek-R1** and **Qwen 2.5/3.5** run locally with near-cloud quality
- **1.58-bit quantization** (BitNet) enables running larger models on modest hardware
- Self-hosting has gone from "niche hobby" to "mainstream strategy" for privacy and cost control
