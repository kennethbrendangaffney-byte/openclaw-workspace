## Executive Summary

The local LLM landscape continues to mature with significant developments in inference engines, quantization methods, and self-hosted solutions. **Major Update (April 2026):** Ollama v0.21.0 introduces Hermes Agent for automatic skill creation, and Google Gemma 4 models are now available with MLX support on Apple Silicon. MLX v0.31.1 brings significant CUDA quantization improvements including 3/5/6-bit support.

### 2. MLX v0.31.1 (Released March 12, 2026)

**Source:** Apple / ml-explore  
**GitHub:** https://github.com/ml-explore/mlx

**Major CUDA improvements:**
- **Quantized GEMV:** Full CUDA implementation for quantized matrix-vector operations
- **3/5/6-bit quantization support:** QMV kernel now supports non-standard bit depths
- **FP16 accumulation:** 4-bit quantization uses fp16 accumulation for better accuracy
- **FSDP support:** CUDA FSDP (Fully Sharded Data Parallel) implementation
- **SegmentedMM:** CUDA implementation for segmented matrix multiplication
- **Hadamard transform:** CUDA implementation for certain quantization schemes

**Apple Silicon improvements:**
- Initial tuning for M5 Pro and Max chips
- Hybrid sharding support for distributed training
- Faster compilation and batch support in QMV

### 3. Llamafile: Single-File LLM Distribution

**Source:** Mozilla AI / Mozilla Ocho  
**GitHub:** https://github.com/mozilla-ai/llamafile

- **What it does:** Allows distributing and running LLMs as a single executable file
- **Impact:** Dramatically simplifies deployment - no dependencies, no Python environment setup

### 4. GGUF Format (llama.cpp)

**Source:** ggerganov/llama.cpp  
**Status:** Merged and established as standard

- **What it is:** Binary format for storing quantized models
- **Advantages:** Efficient loading, supports multiple quantization schemes
- **K-quants:** Advanced quantization (Q4_K_M, Q5_K_M, Q6_K) offering better quality-per-bit than legacy methods
- **Industry adoption:** HuggingFace, Ollama, and most local inference tools now support GGUF

## Quantization Techniques Overview

### Current State of the Art

| Method | Bits | Quality | Speed | Use Case |
|--------|------|---------|-------|----------|
| Q3_K_S | 3 | Acceptable | Fastest | Extreme compression |
| Q4_0 | 4 | Fair | Fastest | Minimal RAM |
| Q4_K_M | 4 | Good | Fast | Balanced (recommended) |
| Q5_K_M | 5 | Very Good | Fast | Quality priority |
| Q6_K | 6 | Excellent | Moderate | Near-FP16 quality |
| Q8_0 | 8 | Near-lossless | Slower | Maximum quality |
| FP16 | 16 | Lossless | Slowest | Training/fine-tuning |

### K-Quants Explained

- **K-means clustering** applied to weights
- **Imbits** (importance bits) preserve critical weight information
- **Mixed quantization:** Different bit depths for different layers based on importance
- **Result:** Better quality at same bit depth vs. uniform quantization

### New: Non-Standard Bit Depths (MLX)

MLX now supports 3-bit, 5-bit, and 6-bit quantization in addition to traditional powers of 2:
- **3-bit:** Extreme compression for edge devices
- **5-bit:** Sweet spot between Q4 and Q6
- **6-bit:** Near-FP16 quality with 2.67x compression

## Emerging Trends to Watch

1. **Ternary quantization (1.58-bit)** - Experimental but promising extreme compression
2. **Speculative decoding** - Draft model + target model for 2-3x speedup
3. **Continuous batching** - vLLM-style efficient serving
4. **KV-cache quantization** - Reducing memory during long contexts
5. **Multi-modal local models** - Vision + language on consumer hardware
6. **Agentic AI tools** - Hermes-style automatic skill creation
7. **MoE (Mixture of Experts)** - More models using sparse activation (Qwen3.5-35B-A3B, Gemma-4-26B-A4B)
