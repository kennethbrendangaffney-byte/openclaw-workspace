## Key Innovation: Sparse MoE Efficiency

The 35B-A3B uses **Mixture of Experts (MoE)**:
- 256 experts total
- Only **8 routed + 1 shared expert activated per token**
- **3 billion active parameters** during inference
- Inference cost scales with active params, not total

**Translation:** You get the knowledge of a 35B model at the speed/cost of a 3B model.

---

## Architecture

**Hybrid attention mechanism:**
- **Gated DeltaNet** — Linear attention (cheaper, handles bulk processing)
- **Gated Attention** — Standard attention with Grouped Query Attention (GQA) for precision
- Repeating pattern: 10 blocks × [3×(DeltaNet→MoE) + 1×(Attention→MoE)]

**Context length:** 262K native, extensible to 1M tokens via YaRN scaling.

## Always-On Chain-of-Thought

Qwen 3.6 removes the thinking/non-thinking toggle:
- **Reasoning is always active** by default
- Faster conclusions, fewer wasted tokens vs Qwen 3.5
- `enable_thinking: False` API param for direct mode if needed

**New: `preserve_thinking`**
- Retains reasoning traces across multi-turn conversations
- Critical for agent loops (coding agents, tool use)
- Reduces redundant reasoning, improves KV cache efficiency
- First open-weight model with this capability

## Relevance to Ken's Setup

### Current Hardware (Beelink EQ, 20GB RAM, no dGPU)
- ❌ Cannot run 27B or 35B models locally at useful precision
- ✅ Can run 4-bit quantized GGUF via llama.cpp/Ollama (slow, CPU-only)
- ✅ API access is extremely cheap (~$0.30 per 1M tokens input)

### Future Build (128GB RAM, RTX 4070 Ti Super 16GB)
- ✅ Can run **35B-A3B at INT8** (10GB VRAM) — **the sweet spot**
- ✅ Can run **27B at 4-bit quant** (~14-16GB VRAM) — fits barely
- ✅ Can run **Plus API** for heavy tasks, local 35B-A3B for everything else
- ✅ KTransformers enables CPU+GPU hybrid for even larger loads

### Strategic Note
The 35B-A3B is uniquely positioned: **3B active params means it runs at ~3B speed with 35B capability**. On a 16GB VRAM card, this is the best coding model you can self-host. The always-on reasoning + thinking preservation make it ideal for agentic workflows (like OpenClaw).
