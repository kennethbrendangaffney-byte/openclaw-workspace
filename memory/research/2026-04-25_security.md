## 2. Linux Hardening Techniques (2026 Best Practices)

### SSH Lockdown (Critical)
- **Disable password authentication** — use Ed25519 SSH keys only.
- **Disable root login** entirely (`PermitRootLogin no`).
- **Change default port** (22) to something non-standard (e.g., 2222) to eliminate 99% of automated scans.
- **Restrict to specific users**: `AllowUsers deploy`
- **Strong algorithms only**:
  - Kex: `sntrup761x25519-sha512@openssh.com`, `curve25519-sha256`
  - Ciphers: `chacha20-poly1305@openssh.com`, `aes256-gcm@openssh.com`
  - MACs: `hmac-sha2-512-etm@openssh.com`
- **Set idle timeouts**: `ClientAliveInterval 300`, `ClientAliveCountMax 2`
- **Use Fail2Ban**: Ban IPs after 3 failed attempts, progressive banning for repeat offenders.

### Firewall Configuration
- **UFW (Uncomplicated Firewall)** for simplicity:
  ```bash
  sudo ufw default deny incoming
  sudo ufw default allow outgoing
  sudo ufw allow 2222/tcp  # SSH
  sudo ufw allow 80/tcp    # HTTP
  sudo ufw allow 443/tcp   # HTTPS
  sudo ufw enable
  ```
- **Rate limit SSH**: `sudo ufw limit ssh`
- **Docker users beware**: Docker bypasses UFW by default. Set `"iptables": false` in `/etc/docker/daemon.json` or bind containers to `127.0.0.1` only.

### Kernel Hardening
- **AppArmor or SELinux**: Enable mandatory access control (MAC).
  - Ubuntu/Debian: AppArmor (enabled by default, enforce profiles)
  - RHEL/CentOS/Fedora: SELinux (set to `enforcing`)
- **Kernel Self-Protection Project (KSPP)** settings:
  - Enable `CONFIG_STACKPROTECTOR_STRONG`
  - Enable `CONFIG_SLAB_FREELIST_RANDOM`
  - Enable `CONFIG_SHUFFLE_PAGE_ALLOCATOR`
  - Enable `CONFIG_INIT_ON_ALLOC_DEFAULT_ON`
- **Disable IPv6** if not needed (prevents leak vectors):
  ```bash
  net.ipv6.conf.all.disable_ipv6 = 1
  net.ipv6.conf.default.disable_ipv6 = 1
  ```

### Automatic Updates
- **Unattended-upgrades** for security patches:
  ```bash
  sudo apt install unattended-upgrades
  sudo dpkg-reconfigure -plow unattended-upgrades
  ```
- Enable automatic reboots at a scheduled time if needed.

### Docker/Container Security
- **Don't run containers as root**: Specify `user: "1000:1000"`
- **Use read-only filesystems** where possible:
  ```yaml
  read_only: true
  tmpfs:
    - /tmp
    - /run
  ```
- **Drop all capabilities**, add back only what's needed:
  ```yaml
  cap_drop:
    - ALL
  cap_add:
    - NET_BIND_SERVICE
  ```
- **Keep images updated** — use Watchtower or Diun for notifications.
- **Scan images** for CVEs before deployment.

## 4. Personal OPSEC (Operational Security)

### The 7-Layer OPSEC Stack
Based on current best practices for individuals with elevated threat models:

1. **IPv6 Kill Switch** — Disable IPv6 at kernel level to prevent VPN bypass
2. **MAC Address Randomization** — Randomize on every boot (`02:` prefix)
3. **Hostname Rotation** — Change hostname on every boot
4. **Firewall Kill Switch** — Default DROP policy; only allow traffic through tunnel
5. **Hardened Tor** — 30-second circuit rotation, destination isolation, exclude Five Eyes exits
6. **Proxychains** — Force all traffic through Tor SOCKS5 proxy, including DNS
7. **WireGuard VPN** — Full tunnel beneath Tor for 5 layers of separation

### DNS Security
- **Use DNS-over-HTTPS (DoH) or DNS-over-TLS (DoT)** to prevent ISP snooping
- **Block ads/malware at DNS level**: Pi-hole or AdGuard Home (self-hosted)

### Browser Hardening
- **Firefox** with `arkenfox/user.js` or **LibreWolf**
- **uBlock Origin** (essential)
- **CanvasBlocker** or **Chameleon** for fingerprinting protection
- **Disable WebRTC** to prevent IP leaks
- **Use containers** for isolating identities (Facebook Container, Multi-Account Containers)

### Communication
- **Signal** for messaging (gold standard)
- **Session** or **SimpleX** for anonymous messaging
- **ProtonMail** or self-hosted email for sensitive communications

### Backup Strategy (3-2-1 Rule)
- **3 copies** of important data
- **2 different media types**
- **1 offsite/offline copy**
- Encrypt backups before uploading anywhere

## 6. Key Takeaways

- **Security is layers, not a single tool.** If one layer fails, others protect you.
- **Fail closed, not open.** If a security service crashes, traffic should stop, not leak.
- **Automation beats memory.** Security that requires manual steps every boot isn't security.
- **Keep it simple.** Complex security is hard to maintain; simple security is hard to break.
- **Stay updated.** The threat landscape evolves fast — what was secure last year may not be today.
