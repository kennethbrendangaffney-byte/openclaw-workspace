# Security Research Summary — 2026-04-21

## Current Threat Landscape (2026)

### Key Findings from Cognyte LUMINAR 2026 Report
- **AI-powered attacks are now mainstream**: Attackers automated 80-90% of a nation-state espionage campaign using LLMs. AI-generated phishing content hit 82.6% of campaigns.
- **Ransomware surged 27.3% YoY**: 7,809 victims claimed in 2025, with Qilin group leading at 12.8% of attacks. Notably, payments fell 23% as attackers shifted to smaller targets (SMBs).
- **Vulnerability explosion**: Nearly 50,000 new CVEs disclosed in 2025 (avg CVSS 6.6). Linux Kernel had the highest count at 2,257 vulnerabilities.
- **Stolen credentials still #1**: Linked to 22% of breaches. Lumma infostealer alone responsible for 2.2 million dark web listings (~42% of total).
- **Regional patterns**: US = 1/3 of ransomware; Middle East dominated by nation-state activity (56.6%); APAC similar at 67%.

### Emerging Threat Vectors
1. **AI-native ransomware**: End-to-end encryption keys generated through federated learning consensus manipulation — a new class of attack.
2. **Supply chain attacks**: CI/CD pipelines, npm/PyPI packages, GitHub Actions increasingly targeted.
3. **React2Shell (CVE-2025-55182)**: Widely discussed on dark web — major web app vulnerability.

## Linux Hardening Techniques (2026 Best Practices)

### SSH Hardening (Critical — Do First)
- **Key-only auth**: Disable PasswordAuthentication, ChallengeResponseAuthentication
- **Ed25519 keys**: `ssh-keygen -t ed25519` — stronger than RSA, smaller keys
- **Disable root login**: `PermitRootLogin no`
- **Restrict algorithms**:
  - Kex: `sntrup761x25519-sha512@openssh.com,curve25519-sha256`
  - Ciphers: `chacha20-poly1305@openssh.com,aes256-gcm@openssh.com`
  - MACs: `hmac-sha2-512-etm@openssh.com,hmac-sha2-256-etm@openssh.com`
- **Idle timeouts**: `ClientAliveInterval 300`, `ClientAliveCountMax 2`
- **AllowUsers directive**: Restrict to specific users only
- **Fail2Ban**: 3 retries, 1hr ban, progressive banning for repeat offenders

### Firewall Configuration
- **UFW** (Ubuntu): Default deny incoming, allow only required ports
- **Rate limiting**: `ufw limit ssh` — max 6 connections per 30s per IP
- **Nftables** (advanced): Dynamic sets for rate limiting, geo-blocking capability
- **Essential open ports**: 22 (SSH), 80/443 (web), and nothing else unless explicitly needed

### Automatic Security Updates
- **Ubuntu**: `unattended-upgrades` with security-only origins
- **Auto-reboot**: Schedule for 3 AM maintenance window
- **Email notifications**: Configure `Unattended-Upgrade::Mail`
- **RHEL**: `dnf-automatic` with similar configuration

### Kernel Hardening (sysctl)
```
net.ipv4.tcp_syncookies = 1          # SYN flood protection
net.ipv4.conf.all.rp_filter = 1       # IP spoofing protection
kernel.randomize_va_space = 2         # ASLR
kernel.kptr_restrict = 2              # Hide kernel pointers
kernel.yama.ptrace_scope = 2          # Restrict ptrace
fs.suid_dumpable = 0                  # Disable core dumps
kernel.unprivileged_userns_clone = 0  # Prevent container escapes
net.core.bpf_jit_harden = 2           # Harden BPF JIT
```

### File Integrity Monitoring
- **AIDE**: Initialize database, run daily checks via cron, email on changes
- **Auditd**: Monitor /etc/passwd, /etc/ssh/, cron, systemd, firewall configs
- **Lynis**: Run `lynis audit system` for comprehensive scoring

### Service Hardening
- **Systemd security directives**:
  - `NoNewPrivileges=yes`
  - `ProtectSystem=strict`
  - `ProtectHome=yes`
  - `PrivateTmp=yes`
  - `MemoryDenyWriteExecute=yes`
- **Disable unnecessary services**: avahi-daemon, cups, bluetooth, ModemManager
- **Run services as dedicated users**: No login shell, no home directory

### MAC (Mandatory Access Control)
- **AppArmor** (Ubuntu): `aa-enforce` profiles for nginx, sshd, etc.
- **SELinux** (RHEL): Keep in enforcing mode, use `audit2allow` for custom policies

## Self-Hosted Privacy Tools (2026 Stack)

### Core Sovereign Stack
1. **Nextcloud** — Personal Google Workspace replacement (files, photos, calendar, contacts, office suite)
2. **Vaultwarden** — Lightweight Bitwarden server in Rust. All premium features, zero cloud dependency
3. **Pi-hole / AdGuard Home** — Network-wide DNS ad/tracker blocking
4. **Jellyfin** — Private Netflix (no "phone home" unlike Plex)

### Secure Remote Access
- **Tailscale** — Zero-config WireGuard VPN. Devices appear on same LAN regardless of location. No port forwarding needed.
- **Cloudflare Tunnels** — Public domain access without opening router ports
- **WireGuard** — Self-hosted VPN option (faster than OpenVPN, simpler config)

### Additional Privacy Tools
- **Email**: Self-hosted Mailcow or Mail-in-a-Box (advanced — requires strong security knowledge)
- **Social**: Mastodon (ActivityPub), Element (Matrix)
- **DNS**: Unbound or AdGuard Home for DNS-over-HTTPS/TLS
- **Backup**: Restic + Rclone to encrypted offsite storage (Backblaze B2, S3)

### Hardware Recommendations
- **Budget**: Raspberry Pi 5 (~$80) — good for Pi-hole, Vaultwarden, light file sharing
- **Sweet spot**: Used mini-PC (Intel NUC/Beelink, ~$150-300) — 16GB RAM, i5/i7, quiet, efficient
- **Power user**: Custom build or Synology/TrueNAS — 40TB+ storage, AI workloads

## Personal OPSEC Recommendations

### Credential Hygiene
- **Password manager**: Use Vaultwarden (self-hosted) or KeePassXC (offline). Never reuse passwords.
- **2FA everywhere**: Hardware keys (YubiKey) preferred. TOTP authenticator apps as fallback. Avoid SMS 2FA.
- **SSH keys**: Ed25519, passphrase-protected, unique per device

### Browser & Device Hardening
- **Firefox + arkenfox user.js**: Hardened privacy profile
- **Brave**: Built-in ad/tracker blocking, Tor private windows
- **Profile isolation**: Separate profiles for work, personal, banking, shopping
- **uBlock Origin**: Essential ad/tracker blocker (even on "privacy" browsers)
- **Containerization**: Use Flatpak/Snap for untrusted apps, or separate user accounts

### Network-Level Privacy
- **DNS-over-HTTPS/TLS**: Prevent ISP snooping on DNS queries
- **Pi-hole/AdGuard**: Block ads/trackers at network level for all devices
- **VPN/Tailscale**: Encrypt traffic on untrusted networks (cafes, hotels)
- **MAC address randomization**: On mobile devices and laptops

### Backup & Recovery
- **3-2-1 rule**: 3 copies, 2 different media, 1 offsite
- **Encrypted backups**: GPG-encrypt before sending to cloud
- **Test restores**: Quarterly verification that backups actually work
- **Immutable backups**: Write-once storage for ransomware protection

### Physical Security
- **Disk encryption**: LUKS (Linux), FileVault (Mac), BitLocker (Windows)
- **Secure boot**: Enable UEFI Secure Boot where possible
- **TPM/Secure Enclave**: Use for key storage if available
