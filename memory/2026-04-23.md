
# 2026-04-23 - Browser Testing Session

## SUCCESS! Google Chrome Works on Virtual Display
- Chrome .deb from Google installed successfully
- **Critical flag: `--ozone-platform=x11`** (forces X11 instead of Wayland)
- Chrome renders perfectly on virtual display
- Successfully navigated to google.com

## Working Chrome Launch Command
```bash
/opt/google/chrome/google-chrome --no-sandbox --disable-gpu --ozone-platform=x11 --window-size=1200,800
```

## Failed Browsers
- Firefox (snap): No window on virtual display
- Chromium (snap): No window, confinement issues
- Epiphany (apt): Window opens but can't load pages

## Full Browser Automation Workflow (Chrome)
1. Launch Chrome with X11 ozone platform
2. Use xdotool directly for clicks/typing (NOT wrapper scripts)
3. Screenshot via `screenshot.sh | base64 -d > file.png`
4. Navigate with Ctrl+L → type URL → Return

## Navigation Test
- Search engine selection: Used Tab key to navigate to "Set as default" button
- Google homepage: Loaded successfully with cookie consent dialog
- Chrome address bar: Accessible via Ctrl+L

## Key Lesson
Wayland is default on Ubuntu 24.04, but Xvfb only supports X11. Must force `--ozone-platform=x11` for Chrome to render on virtual display.

## Chrome Window Leak Issue (11:05 AM)
- Chrome opened on Ken's REAL desktop (DISPLAY :0) instead of virtual display (:99)
- Ken saw 2 Chrome windows pop up on his screen
- Root cause: Chrome detects active display and defaults to it
- **Solution**: Always use explicit DISPLAY=:99 environment variable

## chrome-vnc.sh Launcher Script Created
- Location: `~/workspace/chrome-vnc.sh`
- Forces DISPLAY=:99 and XAUTHORITY=/tmp/.Xauthority-99
- Includes all required flags: --no-sandbox --disable-gpu --ozone-platform=x11
- Usage: `~/workspace/chrome-vnc.sh [optional-url]`

## Cleanup Commands
```bash
# Kill all Chrome processes
pkill -9 chrome
# Verify clean
pgrep chrome || echo "Clean"
```

- More reliable than coordinate clicking for small UI elements

- Virtual desktop verified clean (6.4K empty screenshot)
- Computer-use capability fully restored on Linux
