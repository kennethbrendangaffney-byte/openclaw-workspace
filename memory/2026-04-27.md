

## Direct Chat Logging Bug — Documented

**Issue:** Karen's direct chat sessions with Ken are not persisting to local memory files.

**Evidence:**
- Ken showed a screenshot of "Karen2.0" in direct chat sending an 8-course Khan Academy plan
- Karen searched all local memory files — no trace of this conversation
- Grepped for key terms ("AI for Education", "JavaScript & the Web") — only found in group chat file where Ken pasted them
- No separate direct chat session file exists for 2026-04-27

**Impact:**
- Karen cannot recall direct chat agreements when context switches to group chat
- Creates confusion about what was actually agreed
- Breaks continuity between private and group conversations

**Possible causes:**
1. Direct chat sessions are not being flushed to disk (session compaction skips them)
2. Direct chat files are being saved to a different location
3. File permissions issue on direct chat vs group chat
4. OpenClaw session isolation is too aggressive

**Action needed:**
- Check OpenClaw configuration for session persistence settings
- Verify direct chat session files are being created
- If not, escalate to OpenClaw configuration fix

**Workaround until fixed:**
- Ken pastes key direct chat agreements into group chat for shared memory
- Karen explicitly notes "from direct chat" when recalling agreements
- Use git-synced files for durable plans (not ephemeral chat)
