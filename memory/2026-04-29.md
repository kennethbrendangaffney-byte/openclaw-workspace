# Discord Agent-to-Agent Chat Fix — 2026-04-29

## Problem
KC (cloud agent) could see Karen's messages, but Karen (local agent) could not see KC's messages in `#group-chat`. One-way visibility.

## Root Cause
Two separate issues on Karen's config:
1. **`allowBots: false` (default)** — OpenClaw's Discord extension drops all bot-authored messages by default
2. **`users` allowlist blocking** — Even after enabling `allowBots: true`, the `users` allowlist on the channel config was set to only allow Ken (`1473462044614463518`). KC's bot ID (`1498774042235240549`) was not listed, so her messages were silently dropped.

## The Fix

### Step 1: Enable `allowBots` at root `channels.discord` level
```json
"channels": {
  "discord": {
    "enabled": true,
    "token": "...",
    "allowBots": true,  <-- HERE, not nested under guilds
    "guilds": { ... }
  }
}
```

### Step 2: Add bot IDs to channel `users` allowlist
Even with `allowBots: true`, if `users` is defined on a channel, ALL senders (human or bot) must be explicitly listed:
```json
"1498801547214065876": {
  "requireMention": false,
  "users": [
    "1473462044614463518",   // Ken
    "1498774042235240549"    // KC bot
  ]
}
```

### Step 3: Restart gateway
```bash
systemctl --user restart openclaw-gateway
```

## Key Learning
`allowBots: true` alone is NOT sufficient if `users` allowlist is defined. Bot IDs must be explicitly added to `users` — OpenClaw applies the allowlist filter BEFORE the bot filter. This is either a quirk or intentional layered security.

## Verification
- Used Discord REST API to fetch messages directly: confirmed KC's messages were in Discord
- Checked OpenClaw logs: no `drop bot message` log, but `Blocked discord guild sender` was the actual drop reason
- After fix: KC's messages appeared in Karen's session immediately

## Result
✅ Bidirectional agent-to-agent chat working in `#group-chat`
✅ Multi-agent Discord setup fully operational
